# Code Debugging Challenge

**Find the Bug. Fix the Logic. Win the Game!**

Think coding is only about writing programs?
**Think again.**

The **Code Debugging Challenge** is an exciting competition designed to test your **analytical thinking, problem-solving skills, and attention to detail**. Instead of writing code from scratch, your task is to **identify errors, fix bugs, and make programs run correctly**.

This event pushes you to think critically, work efficiently under time pressure, and prove your **debugging expertise**.

---

## 📍 Event Overview

* Each team must have **2–3 members**
* A **limited number of teams** will participate
* The event will be conducted in a **supervised environment**
* Participants must **debug the given code within the allotted time**

---

## 🔹 Round Structure

### Round 1: Bug Hunt

* Teams will be given **multiple code snippets** containing logical, syntactical, or runtime errors
* The goal is to **identify and fix as many errors as possible within the time limit**
* Each corrected program will be evaluated based on **correctness and efficiency**
* Teams with the **highest scores** will advance to the next round

---

### Round 2: Debugging Under Pressure

* Qualified teams will receive **more complex programs with hidden bugs**

* Errors may include:

  * logical flaws
  * edge case failures
  * performance issues

* Teams must **correct the code and ensure it passes all test cases**

* **Speed and accuracy** will both play a crucial role in scoring

---

## 🧑‍⚖️ Judging Criteria

* Accuracy of corrections
* Time taken to debug
* Code efficiency
* Successful execution of all test cases
